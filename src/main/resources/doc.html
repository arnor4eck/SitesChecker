📖 SitesChecker — Руководство пользователя

SitesChecker — это приложение для периодического мониторинга доступности сайтов и отслеживания изменений их контента.

📋 Содержание

  1. Интерфейс приложения
  2. Создание задачи мониторинга
  3. Просмотр статистики
  4. Управление задачами
  5. Просмотр логов

🖥️ 1. Интерфейс приложения

Верхняя панель

Кнопка Назначение
➕ Добавить Открывает форму создания новой задачи мониторинга
📋 Список Возвращает к списку всех задач мониторинга
📖 Документация Открывает это руководство пользователя
▶️ Старт/Стоп Запускает или останавливает все задачи мониторинга

Основная область

Отображает список задач мониторинга, форму создания новой задачи или статистику проверок.

Нижняя панель

Содержит область логов и индикатор статуса:

  • Статус: Запущено — мониторинг активен
  • Статус: Ожидание — мониторинг остановлен

➕ 2. Создание задачи мониторинга

1 Нажмите кнопку «Добавить» в верхней панели.
2 Заполните форму:
Поле Описание Пример
Название Имя задачи для удобного распознавания Google Homepage
Ссылка URL сайта (должен начинаться с http:// или https://) https://google.com
Период Числовое значение интервала проверки (должно быть больше 0) 5
Единица времени Единица измерения периода Секунда / Минута / Час
3 Нажмите кнопку «Создать» для сохранения задачи.
4 При необходимости нажмите «Очистить» для сброса полей формы.
⚠️ Важно: Все поля обязательны для заполнения. При ошибке валидации появится сообщение с указанием проблемного поля.
💡 Пример: Для проверки сайта каждые 5 минут:
  • Название: Мой сайт
  • Ссылка: https://mysite.com
  • Период: 5
  • Единица времени: Минута

📊 3. Просмотр статистики

Просмотр списка задач

Нажмите кнопку «Список» для отображения всех задач в виде таблицы:

Колонка Описание
ID Уникальный идентификатор задачи
Название Имя задачи мониторинга
Ссылка URL monitored сайта
Период Интервал проверки
Единица времени Единица измерения периода

Просмотр истории проверок

1 Щёлкните правой кнопкой мыши по задаче в списке.
2 В контекстном меню выберите «Проверки».
3 Откроется таблица с историей проверок:
Колонка Описание
Время проверки Дата и время выполнения проверки
Код ответа HTTP-код ответа сервера (200 = OK, 404 = Not Found, и т.д.)
Изменение Показывает, изменился ли контент страницы с последней проверки
💡 Как работает обнаружение изменений: Приложение вычисляет хеш содержимого страницы и сравнивает его с предыдущим значением. Если хеши не совпадают — контент изменился.

⚙️ 4. Управление задачами

Удаление задачи

1 Щёлкните правой кнопкой мыши по задаче в списке.
2 В контекстном меню выберите «Удалить».
3 Подтвердите удаление в диалоговом окне.
⚠️ Важно: Удаление задачи необратимо. Вся история проверок для этой задачи также будет удалена.

Запуск/Остановка мониторинга

  • Нажмите кнопку «Старт» для запуска всех задач мониторинга
  • Нажмите кнопку «Стоп» для временной остановки мониторинга
💡 Совет: При остановке мониторинга все запланированные проверки приостанавливаются до повторного запуска.

📝 5. Просмотр логов

В нижней части окна отображается область логов:

Тип сообщения Описание
INFO Успешные проверки, запуск/остановка приложения
WARN Проблемы с подключением, таймауты
ERROR Критические ошибки подключения или обработки

Примеры сообщений:

  • Приложение запущено — мониторинг активирован
  • Сайт 'Google Homepage' был проверен — проверка выполнена успешно
  • Сайт 'Мой сайт' был добавлен в список мониторинга — задача создана
  • Приложение готово к работе — приложение инициализировано
